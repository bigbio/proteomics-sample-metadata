== Introduction

This document provides guidelines for annotating general sample metadata in SDRF-Proteomics format. These guidelines apply to samples from any organism (human, animal, plant).

For human-specific metadata (disease staging, comorbidities, treatment history), see link:human-sample-metadata.adoc[Human Sample Metadata Guidelines].

**Version 1.0.0** - 2025-01

== Organism

**Column:** `characteristics[organism]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/ncbitaxon[NCBI Taxonomy (NCBITaxon)]

Use the scientific name in lowercase. The validator will map to the correct ontology term.

|===
|Value |NCBITaxon ID |Description

|homo sapiens |NCBITaxon:9606 |Human
|mus musculus |NCBITaxon:10090 |Mouse
|===

== Organism Part / Tissue

**Column:** `characteristics[organism part]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/uberon[UBERON] for mammals/vertebrates, https://www.ebi.ac.uk/ols4/ontologies/po[Plant Ontology (PO)] for plants, https://www.ebi.ac.uk/ols4/ontologies/fbbt[FlyBase Anatomy (FBbt)] for Drosophila.

Use lowercase for all values. For cell line samples, use `not applicable` or specify the tissue of origin (e.g., `cervix` for HeLa).

|===
|Value |UBERON ID |Description

|blood |UBERON:0000178 |Whole blood
|liver |UBERON:0002107 |Liver tissue
|===

== Age

**Column:** `characteristics[age]`

**Format:** `{Number}{Unit}` where Unit is: `Y` (Years), `M` (Months), `W` (Weeks), `D` (Days).

For age ranges use: `{LowerBound}{Unit}-{UpperBound}{Unit}` (e.g., `40Y-50Y`).

Use `not available` when age is unknown, `not applicable` for immortalized cell lines.

|===
|Age Description |SDRF Value

|40 years old |40Y
|8 weeks old |8W
|===

== Developmental Stage

**Column:** `characteristics[developmental stage]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/uberon[UBERON], https://www.ebi.ac.uk/ols4/ontologies/efo[EFO]

|===
|Value |Description

|adult |Sexually mature organism
|embryonic |Pre-birth/pre-hatching
|===

For model organisms, use specific stages when applicable: `embryonic day 14` (mouse E14), `24 hpf` (zebrafish 24 hours post fertilization).

== Disease

**Column:** `characteristics[disease]`

**Ontology:** https://mondo.monarchinitiative.org/[MONDO Disease Ontology] (preferred), https://www.ebi.ac.uk/efo/[EFO], https://disease-ontology.org/[Human Disease Ontology (DOID)]

For healthy controls, use `normal` (mapped to PATO:0000461). For animal disease models, use the human disease name (e.g., `Alzheimer disease` for APP/PS1 mice).

|===
|Disease |SDRF Value |MONDO ID

|Healthy control |normal |PATO:0000461
|Breast cancer |breast carcinoma |MONDO:0004989
|===

== Cell Type

**Column:** `characteristics[cell type]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/cl[Cell Ontology (CL)]

For cell lines, optionally include the cell type of origin.

|===
|Value |CL ID |Description

|epithelial cell |CL:0000066 |General epithelial cell
|T cell |CL:0000084 |T lymphocyte
|===

For detailed immune cell studies, use specific subtypes: `CD4-positive, alpha-beta T cell` (CL:0000624), `regulatory T cell` (CL:0000815).

== Depletion

**Column:** `characteristics[depletion]`

For blood/plasma samples indicating abundant protein depletion.

Values: `no depletion`, `depletion`, `not applicable`.

== Patient-Derived Xenografts (PDX)

**Column:** `characteristics[xenograft]`

When annotating PDX samples, metadata (age, sex) MUST refer to the original patient, not the host organism.

|===
|source name |characteristics[xenograft] |characteristics[organism] |characteristics[age]

|tumor_001 |not applicable |homo sapiens |65Y
|pdx_001 |pancreatic adenocarcinoma grown in nude mice |homo sapiens |65Y
|===

== Synthetic Peptide Libraries

**Column:** `characteristics[synthetic peptide]`

Values: `synthetic` (sample is a synthetic peptide library), `not synthetic` (biological sample).

For synthetic libraries, most sample metadata can be `not applicable`. The organism MAY be specified if the library was designed from specific species peptides.

== PTM Enrichment

**Column:** `characteristics[enrichment process]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/efo[EFO]

|===
|Value |Description

|enrichment of phosphorylated Protein |Phosphoproteomics enrichment
|not applicable |No PTM enrichment performed
|===

== Best Practices

1. **Use lowercase** for all controlled vocabulary values (except proper nouns in disease names).
2. **Use ontology terms** mapped to MONDO (diseases), CL (cell types), UBERON (anatomy), PATO (phenotypes).
3. **Be consistent** with format across all samples in a dataset.
4. **Document unknowns** using `not available` (unknown) or `not applicable` (not relevant) - never leave cells empty.
5. **Validate before submission** using sdrf-pipelines to check ontology mappings.
