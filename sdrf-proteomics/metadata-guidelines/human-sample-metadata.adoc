== Introduction

This document provides guidelines for annotating human-specific metadata in SDRF-Proteomics format. These guidelines apply to clinical studies and healthy human samples.

For general sample metadata (organism, age, sex, basic disease), see link:sample-metadata.adoc[Sample Metadata Guidelines].

**Version 1.0.0** - 2025-01

== Disease Stage

**Column:** `characteristics[disease stage]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/ncit[NCI Thesaurus (NCIT)]

|===
|Value |NCIT ID |Description

|stage I |NCIT:C27966 |Early localized disease
|stage IV |NCIT:C27971 |Distant metastasis
|===

Use `not available` when stage is unknown, `not applicable` for non-cancer samples.

== TNM Staging

**Column:** `characteristics[TNM staging]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/ncit[NCI Thesaurus (NCIT)]

**Format:** `T{0-4}N{0-3}M{0-1}` where T = tumor size/extent, N = lymph node involvement, M = distant metastasis. Prefix `p` indicates pathological staging.

|===
|Value |Description

|T1N0M0 |Small tumor, no nodes, no metastasis
|pT2pN1M0 |Pathological staging after surgery
|===

== Tumor Grade

**Column:** `characteristics[tumor grade]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/ncit[NCI Thesaurus (NCIT)]

|===
|Value |NCIT ID |Description

|grade 1 |NCIT:C28076 |Well differentiated
|grade 3 |NCIT:C28078 |Poorly differentiated
|===

== Cancer-Specific Classifications

**Breast cancer subtype** (`characteristics[breast cancer subtype]`):

|===
|Value |NCIT ID |Description

|luminal A |NCIT:C53554 |ER+/PR+, HER2-, low Ki67
|triple-negative |NCIT:C71732 |ER-/PR-/HER2-
|===

**Other classifications:** `characteristics[Dukes stage]` for colorectal cancer, `characteristics[Ann Arbor stage]` for lymphoma.

== Pre-existing Condition

**Column:** `characteristics[pre-existing condition]`

**Ontology:** https://mondo.monarchinitiative.org/[MONDO Disease Ontology]

For multiple conditions, use multiple columns (preferred) or semicolon-separated values.

|===
|Condition |SDRF Value |MONDO ID

|High blood pressure |hypertension |MONDO:0005044
|Diabetes |type 2 diabetes mellitus |MONDO:0005148
|===

== Treatment

**Column:** `characteristics[treatment]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/ncit[NCI Thesaurus (NCIT)]

|===
|Value |NCIT ID |Description

|chemotherapy |NCIT:C15632 |Chemical drug treatment
|immunotherapy |NCIT:C15262 |Immune-based treatment
|===

Use `untreated` for no treatment, `not available` when unknown.

== Compound

**Column:** `characteristics[compound]`

**Ontology:** https://www.ebi.ac.uk/chebi/[ChEBI], https://www.ebi.ac.uk/ols4/ontologies/ncit[NCIT]

|===
|Compound |ChEBI ID |Drug Class

|doxorubicin |CHEBI:28748 |Anthracycline chemotherapy
|cisplatin |CHEBI:27899 |Platinum chemotherapy
|===

== Dose

**Column:** `characteristics[dose]`

**Format:** `{value} {unit}` (e.g., `100 mg`, `5 mg/kg`, `10 uM`).

== Treatment Status

**Column:** `characteristics[treatment status]`

Values: `pre-treatment`, `on treatment`, `post-treatment`, `treatment naive`.

== Treatment Response

**Column:** `characteristics[treatment response]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/ncit[NCI Thesaurus (NCIT)]

|===
|Value |NCIT ID |Description

|complete response |NCIT:C4870 |No detectable disease
|progressive disease |NCIT:C18206 |Disease progression
|===

== Individual Identifier

**Column:** `characteristics[individual]`

**Format:** Alphanumeric pseudonymized identifier (e.g., `patient_001`, `P023455`).

IMPORTANT: Never use actual patient names. Use pseudonymized identifiers only. Consider GDPR, HIPAA regulations.

When multiple samples come from the same patient, use consistent identifier:

|===
|source name |characteristics[individual] |characteristics[organism part]

|sample_001 |patient_001 |tumor
|sample_002 |patient_001 |adjacent normal
|===

== Body Measurements

|===
|Column |Format |Example

|`characteristics[body mass index]` |Numeric (kg/m^2^) |25.5
|`characteristics[weight]` |{value} kg |70 kg
|===

== Ancestry Category

**Column:** `characteristics[ancestry category]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/hancestro[HANCESTRO]

|===
|Value |HANCESTRO ID

|European |HANCESTRO:0005
|East Asian |HANCESTRO:0009
|===

Use `not available` when ancestry is unknown.

== Smoking Status

**Column:** `characteristics[smoking status]`

**Ontology:** https://www.ebi.ac.uk/ols4/ontologies/ncit[NCI Thesaurus (NCIT)]

Values: `never smoker` (NCIT:C65108), `former smoker` (NCIT:C65107), `current smoker` (NCIT:C65106).

== Menopausal Status

**Column:** `characteristics[menopausal status]`

Values: `pre-menopausal`, `peri-menopausal`, `post-menopausal`.

== Sampling Time Point

**Column:** `characteristics[sampling time point]`

For longitudinal studies, indicates when sample was collected relative to a reference point.

Values: `baseline`, `week 1`, `post-treatment`, `follow-up`.

== Best Practices

1. **Protect privacy** - Never include identifiable patient information.
2. **Use ontology terms** - Prefer MONDO for diseases, NCIT for staging and treatments.
3. **Be consistent** - Use the same format for all samples in a dataset.
4. **Document unknowns** - Use `not available` rather than leaving cells empty.
5. **Include temporal context** - Specify when measurements were taken relative to treatment.
6. **Ensure regulatory compliance** - Follow GDPR, HIPAA, and other data protection regulations.
