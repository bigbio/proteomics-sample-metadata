== Introduction

This directory contains SDRF templates for specific types of proteomics experiments. Each template extends the core SDRF-Proteomics specification with specialized metadata fields for particular experiment types.

Templates provide:

1. **Documentation**: Detailed guidelines explaining when and why to use each template
2. **Checklists**: Required and recommended metadata columns for each experiment type
3. **Template files**: Pre-formatted TSV files with appropriate column headers
4. **Examples**: Sample SDRF entries showing proper annotation

== Template Structure

Each template directory contains:

- `README.adoc` - Complete documentation including:
  * Abstract and motivation for the template
  * Checklist of required and recommended columns
  * Examples and best practices
  * Validation instructions
- `{template-name}-template.sdrf.tsv` - Template file with column headers

== How to Use Templates

=== Choosing a Template

1. **Identify your experiment type**: Determine which specialized template matches your experiment (e.g., single-cell, immunopeptidomics)
2. **Read the documentation**: Review the template's README.adoc to understand requirements
3. **Check the checklist**: Ensure you can provide the required metadata columns
4. **Download the template file**: Use the TSV file as a starting point

=== Using a Template

1. **Copy the template file**: Create a copy of the template TSV for your project
2. **Fill in required columns**: Populate all required columns as documented in the checklist
3. **Add recommended columns**: Include recommended columns where applicable
4. **Validate your SDRF**: Use sdrf-pipelines to validate against the template

=== Validation

All templates can be validated using the sdrf-pipelines tool:

[source,bash]
----
pip install sdrf-pipelines
parse_sdrf validate-sdrf --sdrf_file your_file.sdrf.tsv --template <template-name>
----

Where `<template-name>` is one of: `default`, `human`, `cell-line`, `vertebrates`, `invertebrates`, `plants`, `single-cell`, `immunopeptidomics`, `crosslinking`, `affinity-proteomics`, `metaproteomics`.

== Adding New Templates

To propose a new template:

1. Create an issue on the GitHub repository describing the experiment type and its specific metadata requirements
2. Develop a draft template following the established structure:
   * README.adoc with all standard sections (Status, Abstract, Motivation, Checklist, Examples, Best Practices, etc.)
   * Template TSV file with column headers
3. Create a pull request against the `dev` branch

== Versioning

Following the versioning proposal:

* **Major release** (X.0.0 -> Y.0.0): Changes to the Main SDRF-Proteomics Specification
* **Minor release** (X.Y.0 -> X.Y+1.0): New templates or significant template updates
* **Patch release** (X.Y.Z -> X.Y.Z+1): Refinements to existing templates
