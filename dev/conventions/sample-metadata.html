1. Status of this Convention

This document provides guidelines for annotating general sample metadata in SDRF-Proteomics format. These conventions apply to samples from any organism (human, animal, plant).

Version 1.0.0 - 2025-01

2. Abstract

Sample metadata conventions define standardized formats for common properties that apply across all organism types. This convention covers:

  • Organism annotation

  • Organism part / tissue annotation

  • Age and temporal information

  • Biological sex

  • Disease and health status

  • Developmental stage

  • Cell type annotation

For human-specific metadata (disease staging, comorbidities, treatment history, demographics, lifestyle factors), see Human Sample Metadata Convention.

4.4. Cell Lines and Cell Culture

For cell line samples, use:

  • not applicable - when the sample is a cell line without clear tissue origin context

  • Or specify the tissue of origin (e.g., cervix for HeLa)

5. Age Annotation

Age annotation applies to samples from humans, animals, and other organisms with defined lifespans.

5.1. Column

characteristics[age]

5.2. Ontology

No specific ontology. Use standardized format below.

5.3. Standard Format

{Number}{Unit}

Where Unit is:

  • Y = Years

  • M = Months

  • W = Weeks

  • D = Days

5.4. Examples

Age Description SDRF Value Use Case

40 years old

40Y

Human adult, aged mouse

2 years and 6 months

2Y6M

Human child, dog

8 weeks old

8W

Mouse, rat (common lab age)

10 weeks old mouse

10W

Standard lab mouse age

3 days old

3D

Neonatal samples, zebrafish larvae

Unknown age

not available

When age cannot be determined

5.5. Age Ranges

When exact age is not available but a range is known:

{LowerBound}{Unit}-{UpperBound}{Unit}

Examples:

  • 40Y-50Y - Human between 40 and 50 years

  • 50Y-82Y - Elderly cohort range

  • 8W-12W - Mouse between 8 and 12 weeks

5.6. Missing Values

Value When to Use

not available

Age was not recorded or is unknown

not applicable

Age concept doesn’t apply (e.g., immortalized cell lines)
Important
 Always use not available or not applicable - never leave the field empty.

11. Derived Samples

11.1. Patient-Derived Xenografts (PDX)

In cancer research, patient-derived xenografts (PDX) are commonly used. In these experiments, the patient’s tumor is transplanted into another organism, usually a mouse.

11.1.1. Column

characteristics[xenograft]

11.1.2. Metadata Rules

Important
 When annotating PDX samples, the metadata such as age and sex MUST refer to the original patient and not the host organism (mouse).

11.1.3. Format

The value should describe the growth condition:

Example Value Description

pancreatic cancer cells grown in nude mice

PDX from pancreatic cancer patient

breast tumor grown in NSG mice

PDX from breast cancer patient

not applicable

Non-PDX samples

11.1.4. Linking PDX to Original Tumor

For experiments where both the PDX and the original tumor are measured:

Column PDX Sample Original Tumor Sample

characteristics[xenograft]

Description of growth condition

not applicable

characteristics[source name]

Reference to tumor sample’s source name

not applicable

characteristics[individual]

Patient identifier

Patient identifier (same as PDX)

11.1.5. Example

source name characteristics[individual] characteristics[xenograft] characteristics[organism] characteristics[age] characteristics[sex]

tumor_001

patient_A

not applicable

homo sapiens

65Y

male

pdx_001

patient_A

pancreatic adenocarcinoma grown in nude mice

homo sapiens

65Y

 Both tumor and PDX samples reference the same patient using characteristics[individual]. The age and sex refer to the original patient, not the host mouse.

14. Best Practices

  1. Use lowercase: All controlled vocabulary values should be lowercase (except proper nouns in disease names).

  2. Use standard age format: Prefer {Number}{Unit} format (e.g., 45Y, 10W, 3D).

  3. Use ontology terms: Map values to MONDO (diseases), CL (cell types), UBERON (anatomy), PATO (phenotypes).

  4. Be consistent: Use the same format for all samples in a dataset.

  5. Document unknowns: Use not available (unknown) or not applicable (not relevant) - never leave cells empty.

  6. Use specific terms: Prefer specific ontology terms over generic ones when the information is available.

  7. Validate before submission: Use sdrf-pipelines to validate ontology mappings.
