1. Status of this Template

This document provides a template for annotating Data-dependent acquisition (DDA) mass spectrometry experiments in SDRF-Proteomics format. This template defines additional columns that can be combined with any core template (human, vertebrates, etc.) to capture DDA-specific metadata.

Type: Specialized Template (used in combination with core templates)

Status: Draft - This template is under active development and may change significantly.

Version: 1.0.0-dev - 2025-01

2. Abstract

Data-dependent acquisition (DDA) is the traditional and most widely used mass spectrometry acquisition strategy for proteomics. In DDA mode, the mass spectrometer:

  • Performs a survey scan (MS1) to detect precursor ions

  • Selects the most intense precursors (top-N) for fragmentation

  • Acquires MS2 spectra for selected precursors

  • Uses dynamic exclusion to avoid resampling the same ions

DDA remains the gold standard for discovery proteomics, peptide identification, and building spectral libraries. This template captures the acquisition parameters essential for DDA data analysis and reproducibility.

3. Motivation

DDA experiments have specific parameters that affect data quality and reproducibility:

  1. Top-N selection: Number of precursors selected per cycle

  2. Dynamic exclusion: Duration and settings to avoid resampling

  3. Isolation width: Width of the precursor isolation window

  4. Intensity threshold: Minimum intensity for precursor selection

  5. Fractionation: DDA experiments often use extensive fractionation for deep proteome coverage

  6. Charge state filtering: Which charge states are selected for fragmentation

These aspects require documentation beyond standard SDRF-Proteomics fields.

4. Connections to Analysis Pipelines

DDA data is typically analyzed using:

  • MaxQuant: Widely used for label-free and labeling quantification

  • Proteome Discoverer: Thermo Fisher’s commercial platform

  • MSFragger/FragPipe: High-speed database searching

  • Comet/Percolator: Open-source search and validation

  • Mascot: Commercial database search engine

  • PEAKS: De novo sequencing and database search

  • OpenMS: Open-source proteomics pipeline

5. Additional Ontologies

In addition to the ontologies supported by the core SDRF-Proteomics specification, DDA templates utilize:

  • PSI-MS CV: For mass spectrometry acquisition terms

  • PRIDE CV: For proteomics data acquisition methods and fractionation

8. DDA Acquisition Parameters

8.1. Top-N Selection

The number of precursors selected per MS1 scan significantly affects:

  • Proteome depth

  • Quantification accuracy

  • Duty cycle/cycle time

Top-N Value Typical Use Case Trade-offs

10-15

Standard discovery

Good depth and quantification

20-40

Deep profiling, fractionated samples

Maximum identifications, longer cycle

5-10

Short gradients, targeted

Fast cycles, lower depth

12. Best Practices for DDA Annotation

  1. Always specify acquisition method: Use comment[proteomics data acquisition method] with the PRIDE CV term for DDA.

  2. Document fractionation: Include comment[fractionation method] and properly number fractions with comment[fraction identifier].

  3. Report mass tolerances: Precursor and fragment mass tolerances are essential for data reanalysis.

  4. Include key acquisition parameters: Top-N, isolation window, and collision energy significantly affect results.

  5. Note dynamic exclusion: This parameter affects reproducibility between technical replicates.

  6. Specify MS2 analyzer: Important for understanding resolution and mass accuracy of fragment spectra.

  7. Document fragmentation method: HCD, CID, ETD, and EThcD produce different fragmentation patterns.

  8. Consider instrument-specific settings: Different platforms may have unique parameters worth documenting.
